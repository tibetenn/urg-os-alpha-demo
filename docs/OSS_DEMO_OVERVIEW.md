# URG-OS — OSS Public Demo Overview

## URG-OS とは

URG-OS は **グラフネイティブな Code Governance AI** です。

リポジトリのコード構造を「グラフ」として表現し、依存関係・構造パターン・レビュー時に注目したい領域を可視化します。コードを書き換えたり、自動マージしたりする AI ではありません。

**「いまのコードがどういう状態か」を構造的に把握するための観察・分析ツール**です。

---

## 現時点の Alpha Demo で表現するもの

現時点の Alpha Demo では、主に静的解析をもとに以下の構造情報を抽出します。

| 種別 | 内容 |
|-----------|------|
| **Code Graph** | 関数・クラス・モジュール・import・呼び出し関係などのコード構造 |
| **Security-oriented Structure** | 認証境界・外部入力・シークレット参照など、セキュリティレビュー時に注目したい構造候補 |
| **Runtime-oriented Structure** | 設定依存・副作用・グローバル状態など、実行時のふるまいに影響しうる静的構造 |

現時点の公開デモでは、実行時トレースや本番環境のログを取得しているわけではありません。  
あくまで、公開リポジトリの静的コード構造から「実行時に影響しうる構造」を推定・整理しています。

---

## データの流れ

```text
公開 OSS repo (GitHub)
  └─ clone → AST 解析 → Node / Edge 抽出
       └─ Code Graph / Structure Summary
            └─ Pattern Store (集約・匿名化)
                 └─ Code Governance Report (Markdown + JSON)
                      └─ Privacy Safety Scan → デモパッケージ
```

**raw コードはいかなる段階でも保存されません。**  
レポートに残るのは、ノード型・エッジ型・件数・集約統計などの構造情報です。

## 1000 repo 検証の背景

URG-OS の pipeline が実用規模で安定して動作するかを確認するため、公開 Python OSS リポジトリ 1,000 件を対象に段階的な検証を実施しました。

| ステージ | 規模 | 目的 |
|---------|------|------|
| smoke | 20 repo | 基本動作確認 |
| validation | 100 repo | レポート生成と Safety Scan の確認 |
| stress | 500 repo | スケーラビリティ検証 |
| extended | 1,000 repo | 大規模入力での安定性・安全性確認 |

## プライバシー設計の基本方針

URG-OS は最初から「識別情報を保存しない」という設計原則を持っています。

保存しないもの:

- raw コード
- ファイルパス
- リポジトリ名 / オーナー名
- API エンドポイント
- シークレット名
- メールアドレス
- 実コード由来の関数名・クラス名・qualified name

保存されるもの:

- ノード型
- エッジ型
- 構造上の件数
- 集約統計
- 匿名化された構造パターン

1,000 repo 検証では `privacy_violations: 0` を確認しています。  
Safety Scan は各ステージで自動実行され、すべて `passed: true` で完了しています。

## Code Governance AI の位置づけ

世の中には「コードを自動生成する AI」「PR を自動生成する AI」が増えています。URG-OS はそれらの代替というより、**それらと組み合わせて使うための土台**を目指しています。

自動コーディング AI が「何を書くか」を決める前に、

> いまのコードはどういう構造を持っているのか

を人間が理解できる形で提示すること。  
それが Code Governance AI としての役割です。

---

## 現時点での制限

- 対応言語は主に Python です
- 実行時ログや本番トレースは取得していません
- 脆弱性や品質問題を断定するものではありません
- 自動修正・PR生成・ファイル書き換えは行いません
- Security-oriented / Runtime-oriented な分類は、現時点では静的構造にもとづく候補提示です
