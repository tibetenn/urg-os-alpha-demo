# FAQ — 個人開発者・OSS メンテナ向け

---

## URG-OS について

**Q: URG-OS は何をするツールですか？**

A: グラフ構造を使ってリポジトリのコード状態を可視化する Code Governance AI です。依存関係・呼び出し関係・セキュリティや実行時に関係しうる静的構造をグラフとして整理し、レポートとして出力します。

コードを自動修正したり、PR を生成したりする機能はありません。「観察して報告する」ことに特化しています。

---

**Q: GitHub Copilot や Cursor と何が違いますか？**

A: GitHub Copilot や Cursor は、主に「次に書くコードを提案する」ツールです。URG-OS は「いま存在するコードの構造を把握する」ツールです。用途が異なります。

将来的には、URG-OS が提供するグラフ情報をベースに、自動コーディング AI やレビュー支援ツールがより安全に動作する、という組み合わせを想定しています。

---

**Q: Python 以外のリポジトリは対応していますか？**

A: 現時点では Python (3.6+) のみです。他言語対応は将来のフェーズで検討します。

---

## データとプライバシーについて

**Q: 私のコードは保存されますか？**

A: 保存されません。pipeline は以下の順序で動作します。

1. リポジトリを一時ディレクトリに clone
2. AST 解析でグラフを構築
3. 一時ディレクトリを削除
4. ノード型・エッジ型・件数などの集約情報のみをレポートに保存

raw コード・ファイルパス・関数名・クラス名などは、レポートに含まれません。1,000 件の OSS 検証でも `privacy_violations: 0` を確認済みです。

---

**Q: 診断結果はどこかに公開されますか？**

A: デフォルトでは公開しません。診断結果はあなたにのみ提供されます。  
あなた自身がブログや OSS ドキュメントに掲載したい場合は、もちろん自由に使えます。

---

**Q: 企業がリポジトリの情報を取得しているのではないですか？**

A: URG-OS は現時点で企業製品ではなく、個人が開発している OSS 検証プロジェクトです。企業顧客への販売や、データの第三者提供は行っていません。

---

**Q: LLM にコードを送りますか？**

A: 現時点の OSS Public Demo では、LLM に raw コードを送る設計ではありません。静的解析で得た構造情報をもとに、匿名化・集約されたレポートを生成します。

---

## レポートの内容について

**Q: レポートを読んでもコードのどこが問題か分からないと思うのですが？**

A: それは正しい認識です。URG-OS のレポートは「問題を断定する」ものではなく、「構造を記述する」ものです。

たとえば、レポートが「外部入力から重要処理へつながる可能性のある呼び出し構造」や「認証境界が複雑になりやすい構造候補」を示したとしても、それだけで脆弱性が確定するわけではありません。

URG-OS は、レビューや改善検討の出発点になる構造情報を提示します。最終的な判断は、リポジトリの文脈を知っているメンテナや開発者が行う前提です。

---

**Q: グラフの密度やノード数が多いと、何か悪いということですか？**

A: そうではありません。大規模なリポジトリほど数値は大きくなります。URG-OS は、絶対値だけで良し悪しを判断するのではなく、規模に対してどのような構造の偏りや依存関係があるかを見るための材料を提供します。

現時点では「どんな構造があるか」の把握に主眼を置いています。

---

**Q: 診断を受けたあと、URG-OS に何かをされますか？**

A: 診断後に URG-OS が何かをすることはありません。コードへの書き込みも、外部へのデータ送信も行いません。一時 clone したファイルが削除されれば、pipeline の関与は終わりです。

---

## 利用条件・ライセンスについて

**Q: URG-OS 自体はオープンソースですか？**

A: 現時点では開発中のプロジェクトです。公開範囲とライセンスについては今後のフェーズで決定します。コアアルゴリズムのオープン化を検討しています。

---

**Q: 診断結果を使って URG-OS を宣伝することに同意することになりますか？**

A: なりません。診断を受けることと、URG-OS への言及・推薦は別です。フィードバックを提供してくれる場合は歓迎しますが、義務ではありません。

---

## その他

**Q: フィードバックを送るにはどうすればいいですか？**

A: GitHub Issues が最もシンプルです。「レポートのここが分かりにくい」「この種類の情報が欲しかった」という実用的なフィードバックを特に歓迎します。

---

**Q: URG-OS はいつ本番リリースされますか？**

A: 未定です。現在は検証フェーズ (Phase Alpha) です。1,000 repo の検証が完了し、次は診断品質の改善と、より多様なリポジトリへの対応を進めています。具体的なリリーススケジュールはまだありません。
